## 🔄 The Pipeline (5 Stages + Feedback)

### Stage 1: **Generate Dataset**
- **Onyx (Production)**: Retrieves context from `subjects/reference_docs/{npc}_primer.md`, generates grounded Q&A pairs via RAG
- **Template (Testing Only)**: Fast deterministic generation for smoke tests; never train production on template
- **Output**: `subjects/datasets/{npc}/{technique}/train.jsonl`

### Stage 2: **Sanitize Dataset**
- Validates ChatML format, cleans whitespace, removes empty messages
- **Output**: `.../train_clean.jsonl`

### Stage 3: **Train with Unsloth**
- SFTTrainer with LoRA, config hierarchy: Base YAML < Preset < CLI flags
- **Presets**:
  - `smoke` - debug mode, fast iteration
  - `fast-3b` - standard for RTX 3060 6GB
  - `safe-any` - OOM fallback (use if Ollama running or memory tight)
  - `wandb` - Weights & Biases tracking
- **Output**: `outputs/{npc_key}/` (LoRA adapter) + `exports/{npc_key}/{npc}-lora-f16.gguf`

### Stage 4: **Export & Smoke Test**
- **Adapter Mode (Default)**: Converts LoRA to lightweight GGUF via `convert_lora_to_gguf.py` (MBs, no base needed)
- **Smoke Test**: Validates persona adherence via automated prompts
- **Output**: GGUF ready for llama-server with `--lora` flag

### Stage 5: **Evaluate**
- Starts `llama-server` with `--lora` for adapter evaluation
- Compares baseline vs candidate, generates HTML reports + JSON feedback
- **Output**: `eval/reports/{npc}/eval_YYYY-MM-DD.html`, `eval/results/feedback/{npc}_feedback.json`

### Feedback Loop
- Analyzes eval results → identifies weak concepts → classifies as:
  - `training_density`: Onyx has docs, model didn't learn → regenerate more examples with `--concept-focus`
  - `knowledge_gap`: Onyx has no docs → add reference docs, re-index
- Auto-retrain mode: `./ucore feedback npc.json --auto-retrain --baseline ...`

**⚠️ CRITICAL (6GB VRAM)**: Do NOT use `--auto-retrain` if generating Onyx datasets. Instead:
1. Run generation (`--auto`) first
2. Unload Ollama from memory
3. Manually run training to avoid OOM crashes

## 🛠️ Unified CLI (`ucore`)

### Common Commands
```bash
# Generate
./ucore generate subjects/npc.json --technique onyx                    # Production (Onyx-grounded)
./ucore generate subjects/npc.json --technique onyx --concept-focus dialogue,identity  # Focused

# Sanitize
./ucore sanitize subjects/datasets/npc/onyx/train.jsonl

# Train
./ucore train subjects/npc.json --preset fast-3b                      # Standard
./ucore train subjects/npc.json --preset safe-any                     # Safe OOM fallback
./ucore train subjects/npc.json --preset fast-3b --export-gguf        # Export inline

# Smoke Test
./ucore smoke-test exports/npc/model.gguf

# Evaluate
./ucore evaluate --baseline old.gguf --candidate new.gguf --spec subjects/npc.json --report-html

# Feedback Loop
./ucore feedback eval/results/feedback/npc.json --dry-run             # Preview
./ucore feedback eval/results/feedback/npc.json --auto                # Generate weak concepts only
./ucore feedback eval/results/feedback/npc.json --auto-retrain --baseline old.gguf --train-preset fast-3b  # Full loop

# Audit
./ucore audit check                                                    # Environment + NPC status
```

## 🎓 Key Conventions

### NPC Identity
- **Keys**: `snake_case` (e.g., `history_guide`, `chef_assistant`)
- **Names**: `PascalCase` (e.g., `HistoryGuide`, `ChefAssistant`)
- **System Prompt**: 4-section LLMUnity format (IDENTITY | VOICE | KNOWLEDGE | RULES), ~90-105 tokens

### Dataset Structure
Each NPC trains on **72 total examples**:
| Category | Count | Purpose |
|----------|-------|---------|
| identity | 8 | Who the NPC is (personality, background, mannerisms) |
| teaching | 32 | Subject-matter explanations |
| dialogue | 16 | Natural conversation handling |
| quest | 8 | Scenario-based interactions |
| refusal | 8 | Safe boundary responses |

### GGUF Naming
- **Adapter (LoRA)**: `{npc_key}-lora-f16.gguf` (f16 = float16)
- **Full-Merge**: `{npc_key}-{model_short}-{quant}.gguf` (e.g., `history_guide-3b-q4_k_m.gguf`)

## 🔧 Troubleshooting Quick Reference

### OOM (Out of Memory) on RTX 3060 6GB
1. Check if Ollama is running: `curl http://localhost:11434/api/generate` (kill if running)
2. Use safe preset: `--preset safe-any`
3. Verify Docker allocation: 16-20 GiB recommended for concurrent Supabase + Onyx

### Supabase Connection Issues
- Check status: `supabase status`
- Restart: `supabase stop && supabase start`
- Studio: http://127.0.0.1:16438

### Stale Training Runs
- Check outputs: `outputs/{npc_key}/runs/` (should show 20260517_default_00X)
- Old runs may be archived in `archive/` directory
